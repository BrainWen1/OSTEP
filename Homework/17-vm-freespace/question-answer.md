4. 对 -p BEST 和 -p WORST 都没区别，因为这两不管怎么样都要遍历空闲列表，所以这里只需要对 -p FIRST 讨论。
`./malloc.py -n 10 -H 0 -p FIRST -s 0 -l ADDRSORT -c` 按照块的地址排序
`./malloc.py -n 10 -H 0 -p FIRST -s 0 -l SIZESORT+ -c` 按照块的大小升序排序，即从小到大
`./malloc.py -n 10 -H 0 -p FIRST -s 0 -l SIZESORT- -c` 按照块的大小降序排序，即从大到小

---

5. 不合并的话，会产生大量的外部碎片，仅仅在 -n 100 的时候，最后free list的离散空闲块就有三行；
    但是合并的话，空闲块会大量复用，最后free list只有不到一行。
`./malloc.py -n 100 -H 0 -p FIRST -s 0 -l ADDRSORT -c > no-coalesce.output`
`./malloc.py -n 100 -H 0 -p FIRST -s 0 -l ADDRSORT -C -c > coalesce.output`

---

6. 分配占比更高 -> 空闲块的总量更少 -> 节点数少
如果 -P 99 的话，最后会把空闲块分配完，一直分配失败。
如果 -P  1 的话，几乎alloc和free是一比一的，因为要想调用free，必须要有alloc，不让这就是个逻辑错误，所以要先alloc，然后几乎百分百调用free，以此类推。

---

7. 通过-A连续分配小块，然后间隔释放，就能人为制造出高度碎片化的空闲空间。
`./malloc.py -A +5,+5,+5,+5,+5,+5,+5,+5,-0,-2,-4,-6 -p BEST -l ADDRSORT -C -c`

最后的空闲列表是这样的：
Free List [ Size 5 ]: [ addr:1000 sz:5 ][ addr:1010 sz:5 ][ addr:1020 sz:5 ][ addr:1030 sz:5 ][ addr:1040 sz:60 ]
